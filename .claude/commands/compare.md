# /compare - 챕터 간 비교 분석

대상: $ARGUMENTS

(파일명 2개 지정: "p1-intro.qmd p1-paradigm.qmd", "part8"로 8부 전체, "part9"로 9부 전체, "all"로 전체)

## 목적

챕터 간 내용 중복, 용어 불일치, 분량 불균형, 흐름 단절을 탐지한다. 개별 챕터 리뷰(`/review`)로는 발견하기 어려운 챕터 간 문제를 잡아낸다.

## 분석 차원

### 1. 내용 중복 탐지 (심각도: 높음)

같은 개념을 여러 챕터에서 반복 설명하는 경우를 찾는다.

**탐지 방법:**
- 동일/유사 헤딩 비교
- 동일 코드 예제가 여러 챕터에 등장하는지 확인
- 같은 개념 설명이 2곳 이상에서 나타나는지 확인
- 같은 다이어그램/이미지를 중복 참조하는지 확인

**판단 기준:**
- 동일 내용 반복 → 한 곳에 통합하고 교차 참조
- 관점을 달리한 재설명 → 허용하되 명시적 연결 ("72장에서 살펴본 X를 여기서는 Y 관점에서...")
- 간략한 요약 후 참조 → 허용 (정상적 교차 참조)

### 2. 흐름 분석 (심각도: 높음)

챕터 간 연결이 자연스러운지 점검한다.

**점검 항목:**
- 앞 챕터의 결론이 뒤 챕터의 도입과 연결되는가
- 개념 의존성: 뒤 챕터에서 사용하는 개념이 앞 챕터에서 설명되었는가
- 난이도 곡선이 점진적으로 상승하는가
- 갑작스러운 주제 전환이 없는가

### 3. 분량 균형 (심각도: 중간)

챕터 간 분량이 극단적으로 차이나지 않는지 점검한다.

**기준:**
- 같은 부(Part) 내 챕터 분량 편차: 2배 이내 권장
- 가장 긴 챕터 / 가장 짧은 챕터 비율
- 섹션(##) 수 비교
- 코드/텍스트/시각자료 비율 비교

### 4. 용어 일관성 (심각도: 중간)

챕터 간 같은 개념에 다른 용어를 사용하는지 점검한다.

**점검 항목:**
- 동일 영어 용어의 한글 번역 불일치
- 약어 사용 일관성 (한 챕터에서는 풀어쓰고 다른 챕터에서는 약어)
- 개념 명칭 불일치 (같은 것을 다른 이름으로 부르는 경우)

### 5. 교차 참조 완성도 (심각도: 낮음)

챕터 간 참조가 적절한지 점검한다.

**점검 항목:**
- 관련 내용이 있는데 교차 참조가 없는 곳
- 깨진 교차 참조 (`@sec-` 대상 미존재)
- 과도한 교차 참조 (읽기 흐름 방해)

## 출력 형식

### 두 챕터 비교

```
🔀 챕터 비교: [파일1] vs [파일2]

■ 내용 중복
  [높음] 양쪽 모두 "프롬프트 엔지니어링 기법" 설명
    → 파일1: 라인 45-80 (상세 설명)
    → 파일2: 라인 120-145 (간략 반복)
    → 제안: 파일2에서 제거하고 @sec- 참조로 대체

■ 흐름
  [높음] 파일1 결론부에서 파일2로의 연결 없음
    → 제안: 파일1 마지막에 다음 챕터 예고 문단 추가
  [중간] 파일2에서 "RAG"를 정의 없이 사용 (파일1에서도 미정의)
    → 제안: 파일1 또는 파일2 초반에 RAG 개념 설명 추가

■ 분량
  파일1: 420줄 (## 5개, ### 12개, 코드 8블록, 이미지 3개)
  파일2: 280줄 (## 3개, ### 6개, 코드 4블록, 이미지 1개)
  비율: 1.5:1 (적정 범위)

■ 용어
  [중간] "컨텍스트 윈도우" vs "컨텍스트 창" — 통일 필요

■ 교차 참조
  [낮음] 파일2 라인 90 "앞서 설명한 패러다임" → @sec- 참조 추가 권장

━━━━━━━━━━━━━━━━━━━━━━━━
요약: 높음 X건, 중간 X건, 낮음 X건
```

## 작업 절차

1. `_quarto.yml`에서 챕터 순서를 확인한다.
2. 대상 챕터의 헤딩 구조와 분량을 추출한다.
3. 5가지 차원으로 비교 분석을 수행한다.
4. 심각도 순으로 정렬하여 보고한다.

## 주의사항
- 비교 분석은 읽기 전용 작업이다. 파일을 수정하지 않는다.
- 용어 상세 분석은 `/translate` 스킬로 이어서 작업한다.
- 개별 챕터 심층 리뷰는 `/review` 스킬을 사용한다.
- 수정 작업은 `/write-chapter`로 진행한다.
