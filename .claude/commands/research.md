# /research - 최신 자료 조사

조사 주제: $ARGUMENTS

## 작업 절차

### 1단계: 주제 분석

조사 주제를 파악하고, 관련 챕터를 `PLAN.md`와 `PROGRESS.md`에서 확인한다.
어떤 챕터를 보강하기 위한 조사인지 맥락을 파악한다.

### 2단계: 웹 조사 실행

WebSearch 도구를 활용하여 최신 정보를 수집한다:

- **공식 문서**: 도구/기술의 공식 문서 최신 버전
- **블로그/아티클**: 기술 트렌드, 모범 사례, 사용 후기
- **릴리스 노트**: 최근 업데이트 내용
- **비교 분석**: 유사 도구 간 비교 자료

### 3단계: 정보 정리

조사 결과를 챕터 집필에 바로 활용할 수 있는 형태로 정리한다:

```
📚 조사 결과: [주제]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
핵심 발견
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. [핵심 내용 1]
2. [핵심 내용 2]
3. [핵심 내용 3]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
챕터 반영 제안
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- [어떤 챕터]의 [어떤 섹션]에 [어떤 내용]을 추가
- 새 다이어그램 필요: [설명]
- 용어집 추가 항목: [용어]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
출처
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- [출처 1](URL)
- [출처 2](URL)
```

### 4단계: 후속 작업 제안

조사 결과에 따라 다음 작업을 제안한다:
- `/write-chapter`로 특정 챕터 보강
- `/diagram`으로 새 다이어그램 생성
- `/new-chapter`로 신규 챕터 필요성 판단

## 조사 범위 가이드

| 주제 유형 | 조사 방향 |
|-----------|-----------|
| AI 도구 (Claude Code, Cursor 등) | 최신 기능, 가격, 사용법 변경 |
| 기술 개념 (Context Engineering 등) | 학술/업계 정의, 모범 사례, 사례 연구 |
| IDE/개발 환경 | 최신 버전, 새 기능, 생태계 변화 |
| 트렌드/전망 | 업계 보고서, 전문가 의견, 통계 |

## 주의사항
- 저작권이 있는 콘텐츠는 요약만 하고, 직접 인용은 15단어 이내로 제한한다.
- 출처를 반드시 기록한다.
- 조사 결과는 즉시 챕터에 반영하지 않고, 사용자에게 먼저 보고한다.
- 검증되지 않은 정보는 추가 확인이 필요함을 명시한다.
