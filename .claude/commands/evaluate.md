# /evaluate - 챕터 품질 평가

대상: $ARGUMENTS

(파일명 지정 또는 "all"로 전체 평가, "summary"로 요약 점수표만 출력)

## 평가 기준

5가지 차원에서 각 10점 만점으로 정량 평가한다.

### 1. 내용 충실도 (Content Depth) — 10점

| 점수 | 기준 |
|------|------|
| 9-10 | 핵심 개념 완전 커버, 실전 예제 풍부, 독창적 통찰 포함 |
| 7-8  | 핵심 개념 대부분 커버, 예제 있음, 일부 깊이 부족 |
| 5-6  | 기본 개념 설명은 있으나 예제 부족 또는 피상적 |
| 3-4  | 개요 수준, 빈 섹션 다수 |
| 1-2  | 제목과 골격만 존재 |

**세부 점검:**
- PLAN.md의 해당 챕터 목표 대비 달성률
- 코드 예제의 실용성과 다양성
- 개념 → 실습 → 심화의 학습 곡선
- 최신 기술 동향 반영 여부 (2025-2026 기준)

### 2. 구조 완성도 (Structure) — 10점

| 점수 | 기준 |
|------|------|
| 9-10 | 논리적 흐름 완벽, 적절한 헤딩 깊이, 도입-본문-정리 구조 갖춤 |
| 7-8  | 전체 흐름 양호, 일부 전환 어색 |
| 5-6  | 기본 구조 있으나 흐름 끊김 또는 불균형 |
| 3-4  | 섹션 나열식, 논리적 연결 약함 |
| 1-2  | 구조 미비 |

### 3. 시각 자료 (Visual Assets) — 10점

| 점수 | 기준 |
|------|------|
| 9-10 | 핵심 개념마다 Tufte style SVG 다이어그램 배치, 텍스트-시각 균형 우수 |
| 7-8  | 주요 다이어그램 있음, 일부 텍스트 과다 구간 존재 |
| 5-6  | 다이어그램 1-2개, 대부분 텍스트 위주 |
| 3-4  | 다이어그램 없거나 비-SVG 이미지 사용 |
| 1-2  | 시각 자료 전무 |

**세부 점검:**
- SVG 파일이 Tufte style grayscale 규격 준수하는가
- 다이어그램이 텍스트 내용과 정확히 대응하는가
- 코드/텍스트/다이어그램의 배치 리듬이 적절한가

### 4. 문체 품질 (Writing Quality) — 10점

| 점수 | 기준 |
|------|------|
| 9-10 | 평서문 일관, AI 티 없음, 자연스럽고 간결한 문장 |
| 7-8  | 대체로 양호, 경미한 AI 표현 잔존 |
| 5-6  | 경어체 혼용 또는 AI 패턴 다수 |
| 3-4  | 문체 불일치 심각 |
| 1-2  | 기계 번역 수준 |

**세부 점검:**
- "이 + 명사" 패턴 빈도 (0건이 목표)
- 경어체/평서문 혼용 여부
- 기계적 연결어 빈도
- 문장 길이 편차 (지나치게 긴 문장 비율)

### 5. 기술 정확성 (Technical Accuracy) — 10점

| 점수 | 기준 |
|------|------|
| 9-10 | 코드 실행 가능, API/도구 설명 정확, 버전 정보 최신 |
| 7-8  | 대체로 정확, 사소한 오류 1-2건 |
| 5-6  | 개념 설명 정확하나 코드에 오류 또는 구버전 API 사용 |
| 3-4  | 부정확한 설명 다수 |
| 1-2  | 신뢰할 수 없는 수준 |

## 출력 형식

### 개별 챕터 평가

```
📊 품질 평가: [파일명]

┌─────────────────┬──────┬──────────────────────────┐
│ 평가 차원        │ 점수 │ 코멘트                    │
├─────────────────┼──────┼──────────────────────────┤
│ 내용 충실도      │ 7/10 │ 예제 추가 필요            │
│ 구조 완성도      │ 8/10 │ 도입부 보강 권장          │
│ 시각 자료        │ 5/10 │ 다이어그램 2개 추가 필요   │
│ 문체 품질        │ 9/10 │ AI 패턴 0건              │
│ 기술 정확성      │ 7/10 │ API 버전 업데이트 필요     │
├─────────────────┼──────┼──────────────────────────┤
│ 종합             │ 36/50│ B+                       │
└─────────────────┴──────┴──────────────────────────┘

핵심 개선 사항:
1. [가장 점수가 낮은 차원의 구체적 개선 제안]
2. [두 번째 개선 제안]
3. [세 번째 개선 제안]
```

## 작업 절차

1. `PLAN.md`에서 각 챕터의 목표를 파악한다.
2. `PROGRESS.md`에서 현재 완성도 기록을 확인한다.
3. 대상 `.qmd` 파일을 전체 읽는다.
4. `images/` 폴더에서 해당 챕터의 다이어그램을 확인한다.
5. 5가지 차원별로 점수를 매기고 근거를 기록한다.
6. 종합 점수와 등급을 산출한다.
7. 핵심 개선 사항을 우선순위로 정렬하여 제시한다.

## 주의사항
- 평가는 객관적 기준에 따르되, 점수의 근거를 반드시 명시한다.
- 문체 품질 세부 분석이 필요하면 `/style-check`을 권장한다.
- 평가 결과를 `PROGRESS.md`에 반영하려면 `/update-progress`를 사용한다.
- 동일 챕터를 반복 평가할 때 이전 점수와 비교하여 개선/퇴보를 표시한다.
