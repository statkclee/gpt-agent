# /status - 집필 진행 현황 대시보드

필터: $ARGUMENTS

(인자 없으면 전체, "1부" 또는 "2부" 지정 가능)

## 작업 절차

1. **실시간 분량 측정**: 모든 `.qmd` 파일의 줄 수를 `wc -l`로 측정한다.
2. **PROGRESS.md 읽기**: 기존 기록된 완성도와 비교한다.
3. **PLAN.md 읽기**: 각 Phase별 TODO 완료율을 계산한다.
4. **_quarto.yml 확인**: 목차에 등록된 챕터와 실제 파일 비교 (누락 파일 감지).
5. **이미지 현황**: `images/` 폴더의 파일 수를 센다.

## 완성도 기준

줄 수 기반 완성도 평가 가이드:
- ★☆☆☆☆: ~100줄 (뼈대만 존재)
- ★★☆☆☆: 100~200줄 (기본 프레임워크)
- ★★★☆☆: 200~400줄 (핵심 내용 존재)
- ★★★★☆: 400~700줄 (충실한 내용)
- ★★★★★: 700줄 이상 (완성도 높음)

## 주의사항
- PROGRESS.md와 실제 줄 수가 다르면 불일치를 명시한다.
- _quarto.yml에 없는 .qmd 파일이 있으면 경고한다.
- 마지막 git 커밋 이후 변경된 파일이 있으면 표시한다.
