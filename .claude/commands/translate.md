# /translate - 기술 용어 일관성 관리

대상: $ARGUMENTS

(파일명 지정, "all"로 전체 검사, "glossary"로 용어집 현황, "add 영어:한글"로 용어 등록)

## 목적

기술서 집필에서 영어 용어의 한글 번역이 챕터마다 달라지는 문제를 방지한다. `glossary.qmd`를 기준으로 전체 원고의 용어 일관성을 관리한다.

## 작업 모드

### 모드 1: 용어 일관성 검사 (파일명 / all)

챕터에서 사용된 기술 용어를 추출하고 일관성을 점검한다.

**점검 항목:**
- 같은 영어 용어에 대해 다른 한글 번역 사용 (예: "context" → "컨텍스트" vs "문맥")
- `glossary.qmd`에 등록된 표준 번역과 불일치
- 한글/영어 혼용 불일치 (예: "프롬프트 엔지니어링" vs "Prompt Engineering")
- 약어 사용 불일치 (예: "LLM" vs "대규모 언어 모델" vs "대형 언어 모델")
- 원어 병기 누락 (첫 등장 시 "컨텍스트(context)" 형태 권장)

### 모드 2: 용어집 현황 (glossary)

`glossary.qmd`의 현재 등록 용어를 분석하고, 원고에서 사용되지만 미등록된 용어를 찾는다.

### 모드 3: 용어 등록 (add)

새 용어를 `glossary.qmd`에 추가한다.

## 용어 분류

### 음차 번역 (소리 옮김)
영어 발음을 한글로 옮기는 경우. 기술 커뮤니티에서 음차가 표준인 용어.

| 영어 | 한글 | 비고 |
|------|------|------|
| Prompt | 프롬프트 | |
| Context | 컨텍스트 | "문맥"도 허용하나 기술 용어로는 "컨텍스트" 선호 |
| Token | 토큰 | |
| Agent | 에이전트 | |
| Pipeline | 파이프라인 | |
| Container | 컨테이너 | |
| Framework | 프레임워크 | |

### 의미 번역
의미를 살려 한글로 번역하는 경우.

| 영어 | 한글 | 비고 |
|------|------|------|
| Deployment | 배포 | |
| Debugging | 디버깅 | 음차이나 관용적 |
| Refactoring | 리팩토링 | 음차이나 관용적 |
| Code Review | 코드 리뷰 | |

### 원어 유지
한글 번역 없이 영어 그대로 사용하는 경우.

| 영어 | 비고 |
|------|------|
| API | |
| IDE | |
| LLM | 첫 등장 시 "대규모 언어 모델(LLM)" 병기 |
| RAG | 첫 등장 시 "검색 증강 생성(RAG)" 병기 |
| Git / GitHub | |
| Docker | |
| Quarto | |

## 출력 형식

### 일관성 검사 결과

```
🔤 용어 일관성 검사: [파일명]

■ 불일치 발견
  "context"
    → p1-context.qmd:25 "컨텍스트"
    → p1-context.qmd:88 "문맥"
    → glossary.qmd 표준: "컨텍스트"
    → 제안: 라인 88 "문맥" → "컨텍스트"로 통일

  "large language model"
    → p1-intro.qmd:30 "대규모 언어 모델"
    → p1-workflow.qmd:15 "대형 언어 모델"
    → 제안: "대규모 언어 모델(LLM)"로 통일

■ 원어 병기 누락
  라인 45: "에이전트" 첫 등장 — "에이전트(Agent)" 병기 권장
  라인 72: "RAG" 첫 등장 — "검색 증강 생성(RAG)" 병기 권장

■ 미등록 용어 (glossary.qmd에 없음)
  - "컨텍스트 윈도우" (context window) — 3회 사용
  - "툴 유즈" (tool use) — 2회 사용
  → /translate add "context window:컨텍스트 윈도우"로 등록 가능

━━━━━━━━━━━━━━━━━━━━━━━━
요약: 불일치 X건, 병기 누락 X건, 미등록 X건
```

## 작업 절차

1. `glossary.qmd`를 읽어 표준 용어 목록을 로드한다.
2. 대상 파일에서 기술 용어를 추출한다 (영어 단어, 음차 한글, 약어).
3. 코드 블록, YAML 헤더, 이미지 경로 내 용어는 검사에서 제외한다.
4. glossary 대비 불일치, 챕터 간 불일치, 병기 누락을 탐지한다.
5. 결과를 보고하고 수정을 제안한다.
6. 사용자 확인 후 자동 수정 또는 용어 등록을 실행한다.

## 주의사항
- 코드 내 영어 변수명, 함수명은 검사 대상이 아니다.
- 인용문(> 블록)은 원문 존중으로 검사 제외한다.
- 용어 선택에 정답은 없다. 최종 결정은 사용자에게 맡긴다.
- glossary.qmd 수정 시 알파벳 순서를 유지한다.
