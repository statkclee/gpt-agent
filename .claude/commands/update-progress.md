# /update-progress - 진행 기록 갱신

작업 메모: $ARGUMENTS

## 작업 절차

### 1단계: 변경 사항 감지

```bash
git diff --stat          # 수정된 파일 목록
git diff --name-only     # 변경 파일명만
```

### 2단계: 분량 재측정

모든 `.qmd` 파일의 줄 수를 측정한다:
```bash
wc -l *.qmd
```

### 3단계: PROGRESS.md 업데이트

다음 항목들을 갱신한다:

1. **최종 업데이트 날짜**: 오늘 날짜로 변경
2. **챕터별 분량(줄)**: 실측값으로 갱신
3. **완성도 평가**: 분량 변화에 따라 ★ 재평가
   - ★☆☆☆☆: ~100줄
   - ★★☆☆☆: 100~200줄
   - ★★★☆☆: 200~400줄
   - ★★★★☆: 400~700줄
   - ★★★★★: 700줄 이상
4. **상태 설명**: 내용이 크게 변경된 챕터의 상태 문구 업데이트
5. **이미지 수**: `images/` 폴더 파일 수 갱신
6. **마일스톤**: 완료된 항목 체크, 새 항목 추가
7. **작업 로그**: 오늘 날짜와 작업 내용 추가

### 4단계: PLAN.md 업데이트

1. 완료된 TODO 항목에 `[x]` 체크
2. 작업 중 발견된 새 TODO가 있으면 추가
3. Phase별 진행률 변화 확인

### 5단계: 결과 보고

```
📝 진행 기록 갱신 완료

변경된 챕터:
  📄 73-context-engineering.qmd: 115줄 → 420줄 (★★☆☆☆ → ★★★★☆)
  📄 glossary.qmd: 159줄 → 175줄

PLAN.md 업데이트:
  ✅ Phase 1-1: 컨텍스트 엔지니어링 심화 - 완료

작업 로그 추가:
  2026-02-19 | 73장 컨텍스트 엔지니어링 심화 내용 추가
```

## 주의사항
- PROGRESS.md의 기존 형식을 유지한다.
- 줄 수가 줄어든 경우(내용 삭제)도 정확히 기록한다.
- 전체 진행률 요약(1부 ~60%, 2부 ~65% 등)도 필요시 조정한다.
- $ARGUMENTS가 비어있으면 git diff에서 자동으로 작업 내용을 추론한다.
