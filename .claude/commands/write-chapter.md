# /write-chapter - 챕터 집필/보강

대상 챕터: $ARGUMENTS

## 작업 절차

1. **계획 확인**: `PLAN.md`에서 해당 챕터의 목표와 TODO 항목을 읽는다.
2. **현황 확인**: `PROGRESS.md`에서 현재 완성도와 분량을 확인한다.
3. **기존 내용 분석**: 대상 `.qmd` 파일을 전체 읽고, 현재 구조와 빈 부분을 파악한다.
4. **관련 챕터 참조**: 같은 부(Part)의 다른 챕터를 살펴보고, 내용 중복을 피하고 상호 참조를 고려한다.
5. **집필/보강 실행**: 아래 문체 규칙을 엄격히 준수하며 내용을 작성한다.
6. **진행 기록 업데이트**: 작업 완료 후 `PROGRESS.md`의 해당 챕터 줄 수, 완성도, 작업 로그를 갱신한다.

## 문체 규칙 (CLAUDE.md 기반, 엄격 준수)

### 평서문
- "~합니다" → "~한다" / "~함"
- "~입니다" → "~이다" / "~임"
- 예외: `index.qmd`(서문)만 존칭 허용

### AI 티 제거 (최우선)
- **"이 + 명사" 패턴 금지**: "이 메서드는" → "`startswith()`는", "이 문제를" → "인덱스 오류를"
- **과도한 지시 대명사 금지**: "이러한 방식은" → "해당 방식은" 또는 구체적 명사 사용
- **기계적 연결어 금지**: "다음으로,", "또한,", "마지막으로," 제거
- **불필요한 관형사형 금지**: "~하는 것입니다" → "~한다"

### Quarto 컨벤션
- 코드 블록에 적절한 언어 태그 사용
- 이미지는 `![캡션](../images/파일명.svg)` 형식 (챕터가 `chapters/`에 위치)
- 내부 참조는 `@sec-` 형식 사용
- callout 블록(note, tip, warning, important) 적절히 활용

## 주의사항
- 한 번에 한 챕터만 작업한다.
- 기존 내용을 함부로 삭제하지 않는다. 보강이 원칙이다.
- 새 다이어그램이 필요하면 `/diagram` 스킬 사용을 제안한다.
- 집필 완료 후 `quarto render`로 렌더링 오류가 없는지 확인한다.
