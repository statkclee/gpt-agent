# /render - Quarto 렌더링 및 검증

모드: $ARGUMENTS

(인자 없으면 전체 렌더링, "preview"면 미리보기 모드)

## 작업 절차

### 1단계: 사전 점검

렌더링 전에 기본적인 문제를 먼저 확인한다:

- **_quarto.yml 검증**: YAML 문법 오류, 등록된 파일 존재 여부
- **이미지 참조 확인**: 각 `.qmd` 파일에서 참조하는 이미지가 `images/` 폴더에 존재하는지 확인
- **내부 참조 확인**: `@sec-`, `@fig-`, `@tbl-` 등 cross-reference 대상이 존재하는지 확인

### 2단계: 렌더링 실행

**전체 렌더링:**
```bash
quarto render
```

**미리보기 (preview 모드):**
```bash
quarto preview --port 7771 --no-browser
```

### 3단계: 결과 분석

렌더링 출력을 분석하여 다음을 보고한다:

```
🔨 Quarto 렌더링 결과

상태: ✅ 성공 / ❌ 실패

렌더링된 파일:
  ✅ index.html
  ✅ p1-intro.html
  ...

경고사항:
  ⚠️ [파일]: [경고 내용]

오류 (있을 경우):
  ❌ [파일]: [오류 내용]
  → 원인 분석: ...
  → 해결 방안: ...
```

### 4단계: 후속 조치

- **성공 시**: `PROGRESS.md`의 렌더링 확인 날짜를 갱신한다.
- **실패 시**: 오류 원인을 분석하고, 수정 가능한 문제는 자동 수정을 제안한다.
- **경고 시**: 경고 내용을 사용자에게 보고하고, 심각도에 따라 조치 여부를 확인한다.

## 주의사항
- 렌더링은 시간이 걸릴 수 있다 (타임아웃 10분 설정).
- preview 모드에서는 백그라운드로 실행하고, 사용자에게 URL을 안내한다.
- `docs/` 폴더의 변경사항은 자동 커밋하지 않는다.
- R/Python 코드 실행이 포함된 챕터는 의존성 문제가 있을 수 있다.
