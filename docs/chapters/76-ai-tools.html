6  AI 도구 생태계

AI 코딩 도구는 빠르게 진화하고 있다. GitHub Copilot, ChatGPT, Claude Code, Cursor 등 각 도구는 서로 다른 강점을 가진다. 상황에 맞는 도구를 선택하고 활용하는 것이 중요하다.

6.1 AI 코딩 도구 개요

GitHub Copilot: 인라인 자동완성 - IDE에 통합되어 코드 작성 중 즉시 제안 - 함수명, 주석만 작성하면 구현 자동 생성 - 반복적 패턴에 강점 (CRUD, 보일러플레이트)

ChatGPT: 대화형 코드 생성 - 자연어 대화로 요구사항 정제 - 설명과 함께 코드 제공 - 교육용, 탐색적 코딩에 적합

Claude Code: 컨텍스트 풍부한 협업 - 프로젝트 전체 코드베이스 이해 - 파일 간 관계 파악하여 일관된 코드 생성 - MCP를 통한 도구 통합

Cursor: 코드베이스 전체 이해 - AI가 전체 프로젝트 구조 파악 - 멀티파일 편집 지원 - Composer 모드로 복잡한 작업 수행

6.3 AI Agent: 자율적 코딩 어시스턴트

2025년에는 단순 Chat을 넘어 Agent 모드가 등장했다. 개발자가 “기능 X를 구현해줘”라고 요청하면, Agent가 자율적으로 작업을 수행한다.

Chat 모드 vs Agent 모드:

6.3.1 Cursor Composer

Cursor의 Composer 모드는 복잡한 작업을 자율적으로 수행한다.

# Composer 실행 (Cursor IDE)
Cmd+I (Mac) / Ctrl+I (Windows)

# 요청 예시
"사용자 인증 기능을 추가해줘. JWT 토큰 방식으로."

Composer가 자동으로 수행: 1. 관련 파일 탐색 (auth.py, user.py, routes.py) 2. 여러 파일 동시 수정 3. 테스트 코드 작성 4. 오류 발견 시 자동 수정

개발자는 최종 결과만 검토하면 된다. 멀티파일 리팩토링, 새 기능 추가에 매우 강력하다.

6.3.2 Claude Code Agent

Claude Code는 코드베이스 전체를 이해하는 Agent다.

  • 프로젝트 구조 파악
  • 파일 간 관계 분석
  • MCP로 파일/터미널 제어
  • Git 커밋 자동 생성
# Claude Code 실행
claude code

# 요청 예시
"데이터베이스 스키마를 변경하고 관련 모든 파일을 업데이트해줘"

Claude Code Agent는 영향받는 모든 파일을 찾아 일관되게 수정한다. 개발자는 전체 흐름만 지시하고 세부 사항은 Agent에 위임한다.

Claude Code 커스터마이징:

Claude Code는 Skills와 Slash Commands로 확장 가능하다.

Slash Commands (수동 실행): - 터미널에서 직접 호출하는 단축키 - .claude/commands/ (프로젝트용) 또는 ~/.claude/commands/ (개인용) - 자주 쓰는 프롬프트를 재사용

# 예시: /review 명령어
# .claude/commands/review.md 파일 생성
---
name: review
description: 코드 리뷰
---
이 파일의 코드를 리뷰하고 개선점을 제시해주세요.

# 사용
claude code
> /review

Skills (자동 발견): - AI가 자동으로 발견하고 사용하는 도구 - 대화 맥락에 맞춰 Claude가 선택 - 디렉토리 구조 + 패턴/템플릿 파일 포함

CLAUDE.md: - 프로젝트 루트에 배치 - 항상 로드되는 컨텍스트 (코딩 스타일, 규칙) - .cursorrules와 유사한 역할

커스터마이징 계층: 1. CLAUDE.md: 항상 로드되는 프로젝트 규칙 2. Slash Commands: 수동 호출 (/commit, /review) 3. Skills: 자동 발견 및 사용 4. Subagents: 전문 Agent (별도 컨텍스트)

6.3.3 Aider: CLI Agent

Aider는 터미널에서 실행되는 AI Pair Programmer다.

# Aider 설치
pip install aider-chat

# Aider 실행
aider

# Git 통합 자동
aider --auto-commits

Aider 특징: - Git 통합 (자동 커밋 메시지) - 터미널에서 작업 (IDE 불필요) - 여러 파일 동시 편집 - 변경 사항 diff 표시

사용 예시:

aider> 로그인 기능을 Flask로 구현해줘

# Aider가 자동으로:
# 1. app.py, auth.py, templates/login.html 생성
# 2. requirements.txt에 Flask 추가
# 3. git commit -m "Add Flask login feature"

6.3.4 OpenAI Codex CLI

OpenAI Codex CLI는 OpenAI의 자율적 코딩 Agent다.

# Codex CLI 설치
npm install -g @openai/codex-cli

# Codex 실행
codex

# 요청 예시
codex> "이 프로젝트에 사용자 인증을 추가해줘"

Codex CLI 특징: - GPT-5.2-codex 모델 사용 (2025년) - 터미널에서 자율 실행 - 오픈소스 (Rust로 작성) - macOS/Linux 지원, Windows 실험적

주의: 2021년 Codex API는 2023년 3월 단종되었으나, 2025년 5월 새로운 Codex CLI가 출시되었다. 이는 단순 모델이 아니라 완전한 자율 소프트웨어 엔지니어링 Agent다.

6.3.5 Google Gemini CLI

Google Gemini CLI는 Google의 공식 터미널 AI Agent다.

# Gemini CLI 설치
npm install -g @google/gemini-cli

# Gemini 실행
gemini

# 요청 예시
gemini> "데이터베이스 마이그레이션 스크립트 작성"

Gemini CLI 특징: - Gemini 2.5 Pro 모델 (1M 토큰 컨텍스트) - Google Search 통합 (실시간 정보 검색) - MCP 지원 (커스텀 도구 통합) - Cloud Shell에 기본 탑재

무료 제공: - 개인 Google 계정으로 로그인 - 분당 60회, 일일 1,000회 요청 무료 - Gemini Code Assist 라이선스 포함

6.3.6 Google Antigravity: Agent 우선 IDE

Google Antigravity는 2025년 11월 Gemini 3와 함께 발표된 Agent 우선 개발 플랫폼이다.

# Antigravity는 별도 IDE (다운로드 설치)
# https://antigravity.google

# 사용 예시 (IDE 내에서)
"사용자 인증 기능을 추가하고 테스트까지 작성해줘"

Antigravity 특징: - Agent-First 아키텍처: 작업(Task) 수준 지시 → Agent 자율 실행 - Gemini 3 통합: Pro, Deep Think, Flash 모델 - 비동기 워크플로우: 에디터, 터미널, 브라우저 통합 - 검증 가능성: Agent 작업 과정 추적 및 검증 - 무료 제공: 개인 사용자 대상

비전: “아이디어만 있으면 누구나 소프트웨어를 만들 수 있게”

Antigravity는 Cursor, Claude Code와 경쟁하는 Google의 본격적인 Agent IDE다. 이름의 의미는 “반복 작업, 환경 설정, 디버깅의 중력(Gravity)에서 벗어나기”다.

6.3.7 Agent 사용 시 주의사항

경고Agent의 한계

Agent가 강력하지만 완벽하지 않다.

주의할 점: - 복잡한 작업일수록 오류 가능성 증가 - Agent가 생성한 코드를 반드시 검토 - 보안 취약점은 여전히 45% - Git 자동 커밋은 위험 (리뷰 후 수동 커밋 추천)

권장 사용법: - 간단~중간 복잡도: Agent 사용 - 고도로 복잡한 작업: Chat 모드로 단계별 진행 - 항상 최종 검증은 인간이 수행

6.4 IDE 통합

VS Code 확장 - GitHub Copilot: 자동완성 - Continue: 오픈소스 AI 어시스턴트 - Codeium: 무료 대안

Positron 통합 - R과 Python을 위한 차세대 IDE - AI 어시스턴트 내장 - Jupyter 스타일 인터랙티브 개발

설정과 커스터마이징 - 프롬프트 템플릿 설정 - 컨텍스트 범위 조정 - 단축키 설정

6.5 MCP 서버 활용

MCP(Model Context Protocol)는 Anthropic이 제안한 “AI를 위한 USB-C”다. 표준화된 방식으로 외부 시스템과 연결한다.

주요 MCP 서버: - 파일시스템: 파일 읽기/쓰기 - 데이터베이스: PostgreSQL, SQLite 쿼리 - GitHub: 이슈, PR 관리 - Slack: 메시지 전송

설치 예시:

# Claude Code용 MCP 서버
npm install -g @modelcontextprotocol/server-filesystem

# 설정 파일에 추가
# ~/claude/mcp_settings.json

생각해볼 점

AI 코딩 도구 생태계는 빠르게 진화한다. 2022년 GitHub Copilot에서 시작하여, 2025년에는 Cursor, Claude Code 등 다양한 도구가 등장했다. 각 도구는 서로 다른 철학과 강점을 가진다.

단일 도구에 의존하기보다 상황에 맞게 조합하는 것이 효과적이다. 빠른 자동완성에는 Copilot, 복잡한 리팩토링에는 Cursor, 학습과 탐색에는 ChatGPT를 사용하는 식이다. 도구의 강점을 이해하고 적재적소에 활용하는 감각이 필요하다.

MCP의 등장은 AI 도구 생태계의 표준화를 의미한다. 과거 각 도구가 독자적인 통합 방식을 사용했다면, MCP는 “플러그인 아키텍처”를 제공한다. 한 번 MCP 서버를 설치하면 여러 AI 도구에서 공통으로 사용할 수 있다.

프로젝트

  1. GitHub Copilot을 설치하고, 간단한 함수를 자동완성으로 작성해보라.
  2. ChatGPT에게 동일한 기능을 요청하고, Copilot 결과와 비교하라. 어느 쪽이 더 나은가?
  3. MCP 파일시스템 서버를 설치하고 Claude Code에서 활용해보라.
  4. 각 도구의 강점과 약점을 정리하여 자신만의 도구 선택 가이드를 만들어라.
