6  프롬프트에서 컨텍스트로

2025년 2월, 안드레이 카파시는 “Vibe Coding”이라는 개념을 제안했다[1]. “완전히 분위기에 몸을 맡기고, 기하급수적 성장을 받아들이며, 코드가 존재한다는 것조차 잊어라”라는 선언은 프로그래밍 패러다임의 급진적 전환을 상징한다. Collins Dictionary는 “vibe coding”을 2025년 올해의 단어로 선정했다[2].

Vibe Coding의 핵심은 코드 자체보다 결과에 집중하는 것이다. 개발자는 “이런 기능이 필요해”라고 의도를 설명하고, AI가 작동하는 코드를 생성한다. 전통적 개발에서 요구사항 분석, 설계, 코딩, 테스트, 디버깅을 순차적으로 거쳤다면, Vibe Coding에서는 의도 설명과 결과 확인만 남는다. Microsoft CTO는 2030년까지 95%의 코드가 AI에 의해 생성될 것으로 예측했다.

그러나 Vibe Coding에는 명확한 한계가 있다. Veracode의 2025년 연구에 따르면, AI 생성 코드의 45%에 보안 취약점이 포함되어 있다. “Vibe Coding Hangover”라는 표현이 등장할 정도로, 시니어 개발자들 사이에서 검증되지 않은 AI 코드로 인한 “개발 지옥” 보고가 이어지고 있다. Vibe Coding은 프로토타이핑과 탐색적 개발에 강점을 보이지만, 프로덕션 환경에서는 여전히 인간의 검증이 필수다.

6.0.1 그럼에도 남는 것: 패턴 지식의 새로운 역할

재사용 명령이 약화되었다고 패턴과 아키텍처 지식이 무용해진 것은 아니다. 역할이 변화했을 뿐이다.

“Repository 패턴을 사용했는가?”, “SQL 인젝션 방어가 되어 있는가?”, “트랜잭션 경계가 적절한가?”—이런 질문에 답하려면 패턴과 아키텍처 지식이 필수다. AI가 코드를 생성하더라도, 검증하는 것은 인간의 역할이다. 개발자의 정체성은 Coder(코드 작성자)에서 Orchestrator + Verifier(오케스트레이터 + 검증자)로 전환된다.

6.1 프롬프트 공학에서 컨텍스트 공학으로

프롬프트 공학(Prompt Engineering)은 LLM으로부터 원하는 결과를 얻기 위해 입력 지시문을 설계하는 기술이다. Software 3.0 시대에서 프롬프트는 곧 프로그램이므로, 프롬프트 작성 능력은 핵심 역량이 되었다.

효과적인 프롬프트의 구성요소는 다음과 같다.

역할 지정: LLM에게 특정 전문가 역할을 부여한다. “당신은 시니어 파이썬 개발자입니다”라는 지시가 있으면 코드 품질과 관례 준수 수준이 높아진다.

맥락 제공: 문제의 배경, 제약 조건, 기존 코드 등 관련 정보를 제공한다. 맥락이 풍부할수록 정확한 출력을 얻는다.

명확한 지시: 원하는 결과물의 형식, 길이, 스타일을 구체적으로 명시한다. “코드만 작성하고 설명은 생략해주세요”와 같은 지시가 출력 품질을 높인다.

예시 제공: 입력-출력 쌍의 예시를 제공하면 LLM이 패턴을 파악하여 일관된 결과를 생성한다. 이를 few-shot prompting이라 한다.

생각해볼 점

프롬프트 공학에서 컨텍스트 공학으로의 진화는 개발자 역할 변화를 반영한다. “좋은 질문을 던지는 기술”에서 “AI가 일할 수 있는 환경을 설계하는 기술”로 확장되었다. 단일 프롬프트 최적화에서 시스템 전체의 맥락 흐름 설계로 관점이 이동한 것이다.

컨텍스트 공학의 4대 기둥—시스템 프롬프트, 동적 맥락, 도구 통합, 구조화 출력—은 모두 “AI가 더 나은 결과를 생성하도록 환경을 설계”하는 것이다. CLAUDE.md나 .cursorrules 같은 프로젝트 규칙 파일은 지속적 지침을 제공하고, RAG는 실시간 관련 정보를 주입하며, MCP는 외부 시스템과의 표준화된 연결을 가능하게 한다.

Shopify CEO 토비 뤼트케의 표현처럼 “AI를 위한 완전한 각본을 쓰는 것”이 개발자의 새로운 역할이 되었다. 개별 프롬프트를 잘 쓰는 능력도 중요하지만, 시스템 차원에서 맥락을 설계하는 능력이 더 중요해졌다.

프로젝트

  1. 프로젝트 루트에 CLAUDE.md 파일을 작성하라. 프로젝트 규칙, 코딩 스타일, 제약 조건을 명시한다.
  2. Few-shot prompting 방식으로 함수를 생성하라. 입력-출력 예시 3개를 제공하고 AI가 패턴을 학습하도록 한다.
  3. MCP 서버를 하나 설치하고 Claude Code나 Cursor에서 활용해보라. 파일시스템 또는 데이터베이스 MCP를 추천한다.
  4. 동일한 작업을 “간단한 프롬프트”와 “풍부한 컨텍스트”로 각각 요청하고 결과를 비교해보라.
