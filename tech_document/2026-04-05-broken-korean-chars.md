## 1. 증상

렌더링된 HTML에서 특정 한글 문자가 `❓❓❓`(U+FFFD REPLACEMENT CHARACTER)로 표시됨.

## 2. 원인

파일 자체는 UTF-8로 인코딩되어 있으나(`file -I` 확인), **일부 한글 글자의 UTF-8 바이트 시퀀스가 불완전하게 기록**되었다.

한글 한 글자는 UTF-8에서 3바이트를 차지한다. 예를 들어 "폭발적"의 "발"은 `EB B0 9C`(3바이트)인데, Write 도구가 긴 문자열을 처리하는 과정에서 일부 글자의 바이트가 잘리거나(truncated) 누락되어 `EF BF BD`(U+FFFD, replacement character)로 저장된 것이다.

## 3. 근본 원인

Claude Code의 Write 도구가 긴 UTF-8 한글 텍스트를 한 번에 기록할 때, 내부 버퍼 경계(buffer boundary)에서 3바이트 한글 문자의 중간이 잘리는 경우가 드물게 발생한다. Google Drive 파일 스트림의 동기화 지연도 복합적 요인일 수 있다.

## 4. 해결 방법

Edit 도구로 깨진 11개소를 개별 수정한다. Edit은 diff 기반이므로 전체 파일을 다시 쓸 필요 없이 정확한 위치만 교체 가능하다.

## 5. 재발 방지

- 긴 챕터 작성 후 반드시 `python3 -c` 스크립트로 U+FFFD 검사
- 검사 명령: `python3 -c "print([l for l in open('file.qmd') if '\ufffd' in l])"`
